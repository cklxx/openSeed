# Design: Research OS v1.0

Generated by CEO review on 2026-03-18 | Branch: main | Mode: SCOPE EXPANSION

## Vision

OpenSeed evolves from a research CLI tool into a **research operating system** вҖ” it watches
your field continuously, builds a knowledge graph of connected papers, generates weekly
digests, and closes the loop from research to writing.

The emotional quality: "This tool thinks about my field even when I'm not."

### 12-Month Ideal

Monday morning: you open OpenSeed. It says "3 new papers in your areas this week.
Paper X extends the method from Paper Y you read last month вҖ” here's how they connect."
You see clusters forming in your knowledge graph. You tap "deep dive" on a cluster and
get a synthesized narrative. During writing, you ask "what are the strongest
counterarguments to attention-based approaches?" and get cited, nuanced answers from
your library + fresh search.

## Accepted Scope

### v0.6.0 вҖ” Foundation
1. **SQLite storage migration** вҖ” Replace JSON flat files with SQLite (indexed, concurrent, FTS5).
   Auto-migrate from JSON on first run (backup JSON files first).
2. **Paper knowledge graph** вҖ” Relationship tracking between papers. During summarization,
   AI extracts references + Semantic Scholar validates/enriches edges.
   New commands: `paper graph <id>`, `paper cluster`.

### v0.7.0 вҖ” Proactive Research
3. **Scheduled paper watching** вҖ” Crontab-based scheduling via `watch schedule/unschedule`.
   Generates markdown digest of new papers. Log file + `watch status` for observability.
4. **Similar papers + smart reading queue** вҖ” After summarize, suggest 3-5 related papers
   via Semantic Scholar recommendations API. Smart `paper next` ranks by relevance to
   current research focus.

### v0.8.0 вҖ” Research-to-Writing
5. **Paper comparison** вҖ” `agent compare <id1> <id2>` for structured side-by-side analysis:
   methodology differences, contradictory findings, complementary strengths.
6. **LaTeX related-work export** вҖ” Turn synthesis into a `\section{Related Work}` draft
   with `\cite{}` references and BibTeX entries.

## Key Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Migration strategy | Auto-migrate on first run | Frictionless; data is small; JSON backed up |
| Scheduler | OS crontab | Universal, zero deps, researchers understand "runs daily" |
| Graph edge detection | AI extraction + Semantic Scholar | AI catches informal refs, S2 gives ground truth |
| Release strategy | Incremental (v0.6, v0.7, v0.8) | Smaller blast radius, feedback loops |
| Cron safety | Fixed command, no interpolation | No user input in crontab strings (injection prevention) |
| Cron observability | Log file + `watch status` command | Cron fails silently by default; must be observable |
